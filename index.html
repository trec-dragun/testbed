Lateral-reading skill

A portable research workflow for checking claims beyond the target article.

The skill asks an agent to leave the source text, search laterally, compare independent evidence, and return a structured trustworthiness report with citations.

Corroborate central claims

The workflow prioritizes independent sources that confirm, qualify, or dispute the article's important factual claims rather than summarizing the article in isolation.

Trace source quality

Reports distinguish primary sources, official records, reputable reporting, and weaker evidence, making the evidentiary basis visible to readers and evaluators.

Return auditable artifacts

Each run produces JSON and rendered HTML reports with explicit citations, allowing downstream scoring, inspection, and reuse by benchmark participants.

Experimental setting

Useful empirical evidence, not a controlled model ranking.

The benchmark is designed to make agent behavior and report artifacts inspectable. The current leaderboard should be read as one run under one harness and date.

For this public run, Codex executed the lateral-reading skill against anonymous DRAGUN articles. OpenRouter supplied the generation models: GPT-5.5, Claude Opus 4.7, and Gemini 3.1 Pro Preview. AutoJudge then scored the produced reports using GLM-5.1 through OpenRouter.

The setup intentionally keeps private evaluation metadata outside the model-facing workspace and validates output schema, citation URLs, and forbidden-source leakage before leaderboard reporting.

It is not a strict controlled experiment. Model provider routing, backend batch behavior, service-tier scheduling, OpenRouter web search and fetch behavior, search index freshness, model nondeterminism, and AutoJudge variance can all influence results.
